# คู่มือ API: ช่วงลงทะเบียนเรียนและเพิ่ม-ถอน
(`MIS_API_REG_ENROLL_PKG`)

## 📌 1. ภาพรวมของระบบ (System Overview)

Package นี้คือ **"Core Engine ตัวจริง"** สำหรับช่วงลงทะเบียนเรียนปกติและช่วงเพิ่ม-ถอน ความแตกต่างที่สำคัญที่สุดจากระบบแจ้งความประสงค์ (Pre-Reg) คือ **การทำงานแบบ Real-time** หากที่นั่งเต็มคือเต็มทันที และเมื่อกดยืนยัน จะเกิดภาระหนี้สินจริงขึ้นในระบบการเงิน

**โครงสร้างข้อมูลหลัก (Core Tables):**

- `AVSREG.SUBMITITEM`: ตะกร้าพักรายวิชา (Cart) ชั่วคราวของนักศึกษา (เทียบเท่า `PRE_REG_DTL`)
    
- `AVSREG.ENROLLSUMMARY`: ตารางเก็บข้อมูลการลงทะเบียนเรียนที่ยืนยันแล้ว (Official Enrolled Courses)
    
- `AVSREG.ENROLLFEE`: ตารางเก็บภาระหนี้สิน/ค่าธรรมเนียมที่เกิดขึ้นจริง
    
- `AVSREG.TRANSCRIPT`: ตารางประวัติการศึกษา (ใช้เช็คเงื่อนไขรีเกรด และ W)
    

**รูปแบบการคืนค่า (Return Status):**

- `200` = สำเร็จ (Success)
    
- `403` = ติด Business Logic (ที่นั่งเต็ม, สอบชน, หน่วยกิตเกิน/ขาด)
    
- `507` = Exception (ระบบขัดข้อง)

## ⚙️ 2. สรุป Business Logic และฟังก์ชันการทำงานหลัก

### กลุ่มที่ 2.1: การจัดการตะกร้าลงทะเบียน (CRUD Operations on `SUBMITITEM`)

ฟังก์ชันกลุ่มนี้จะทำงานเมื่อนักศึกษากด Add / Drop บนหน้าจอเว็บ (ข้อมูลยังอยู่ในตะกร้า ยังไม่สมบูรณ์จนกว่าจะกดยืนยัน)

- **`ENROLL_INS` (เพิ่มรายวิชาเข้าตะกร้า)**
    
    - **Business Logic 🚨:** ด่านแรกที่ดุเดือดที่สุด ระบบจะเช็ค **จำนวนที่นั่งคงเหลือ (Seat Available)** แบบ Real-time ทันที หาก `ENROLLSEAT >= TOTALSUMSEAT` ระบบจะบล็อกการ Add ทันที (Error 5002 รับเต็มแล้ว)
        
    - เช็คตารางสอบและตารางเรียนซ้ำซ้อน (Exam/Time Conflict)
        
    - เช็คสถานะรายวิชาว่าเปิดให้ลง (Open) หรือไม่
        
- **`ENROLL_DEL` (ถอนรายวิชา)**
    
    - **Business Logic 🚨:** มีความซับซ้อน 2 ระดับ:
        
        1. **ช่วงเพิ่ม-ถอนปกติ:** ถอนได้เลยหากหน่วยกิตไม่ต่ำกว่าเกณฑ์ขั้นต่ำ (Credit Min) รายวิชาจะหายไป
            
        2. **ช่วงถอนติด W:** หากเลยกำหนดเวลาที่ให้ถอนปกติ ระบบจะบังคับให้เป็นการถอนแบบติด "W" (Withdrawal) ซึ่งรายวิชาจะยังค้างอยู่ใน `ENROLLSUMMARY` แต่สถานะเกรดจะกลายเป็น W และการคิดเงินคืน (Refund Rate) จะเปลี่ยนไปตามประกาศของมหาวิทยาลัย
            
- **`ENROLL_CHANGE_SECTION` (เปลี่ยนกลุ่มเรียน)**
    
    - การย้าย Section จะต้องเช็คที่นั่งของ Section ใหม่เสมอ หาก Section ใหม่เต็ม จะย้ายไม่ได้
        

### กลุ่มที่ 2.2: การยืนยันการลงทะเบียน (Confirmation & Master Validation)

- **`ENROLL_CONFIRM` (กดยืนยันการลงทะเบียน)**
    
    - **Business Logic 🚨:** นี่คือจุดที่หนักที่สุดของระบบ API จะทำการเรียกใช้ Procedure ยักษ์ใหญ่ (เช่น `MFU_API.MAINREGISTER`) เพื่อตรวจสอบเงื่อนไขขั้นสุดท้ายทั้งหมด:
        
        - **กวาดข้อมูลจาก `SUBMITITEM`** มาเช็ค Prerequisite, Check Re-grade, Check Credit Max อีกครั้ง (ป้องกันกรณีเด็กเปิด 2 หน้าจอแล้วกดยืนยันพร้อมกัน)
            
        - **ย้ายข้อมูล:** ถ้ายืนยันผ่าน ข้อมูลจะถูกย้ายจาก `SUBMITITEM` เข้าสู่ตาราง `ENROLLSUMMARY` แบบถาวร
            
        - **คำนวณการเงิน:** ระบบจะเรียกสูตรคำนวณค่าเทอม (ตาม `FEEGROUPID`) และ Insert หนี้สินลงใน `AVSREG.ENROLLFEE`
            

### กลุ่มที่ 2.3: การดึงข้อมูล (Data Fetching)

- **`ENROLL_SEL`:** ดึงรายวิชาที่อยู่ในตะกร้า (`SUBMITITEM`) และวิชาที่ยืนยันแล้ว (`ENROLLSUMMARY`) มาวาดแสดงผลบนหน้าจอ
    
- **`ENROLL_CREDIT_SEL`:** คำนวณสรุปยอดหน่วยกิตทั้งหมด เพื่อแสดงให้นักศึกษาดูว่าลงไปกี่หน่วยกิตแล้ว
    
- **`GET_ENROLL_FEE`:** ดึงข้อมูลจาก `ENROLLFEE` มาสรุปยอดเงินที่ต้องชำระในเทอมนั้นๆ



 
## Procedure Documentation : `MFU_API.MAINREGISTER_SUBMIT`

# 📖 Overview  
  
`MFU_API.MAINREGISTER_SUBMIT` เป็น **Oracle PL/SQL Stored Procedure** ที่ใช้สำหรับ  
  
> การ **Submit และตรวจสอบการลงทะเบียนเรียนของนักศึกษา**  
  
Procedure นี้เป็น **Core Logic ของระบบลงทะเบียน**    
โดยจะทำหน้าที่ตรวจสอบข้อมูลก่อนยืนยันการลงทะเบียน เช่น  
  
- สถานะนักศึกษา  
- รายวิชาที่เลือก  
- เวลาที่อนุญาตให้ลงทะเบียน  
- เงื่อนไขการควบคุมระบบ  
- ความถูกต้องของรายการวิชา  
  
หากเกิด Error ระบบจะส่งค่ากลับผ่าน  

VERRID  
VERRSTR

  
และบันทึกลงในตาราง  

AVSREG.SUBMIT

  
---  
  
# ⚙️ Procedure Signature  
  
```sql  
PROCEDURE MFU_API.MAINREGISTER_SUBMIT(  
    VSTUDENTID      IN NUMBER,  
    VTYPE           IN VARCHAR2,  
    VMETHOD         IN VARCHAR2,  
    VWAUTO          IN VARCHAR2,  
    VERRID          IN OUT NUMBER,  
    VERRSTR         IN OUT VARCHAR2,  
    VSUBMITUSERID   IN VARCHAR2,  
    VSUBMITIP       IN VARCHAR2,  
    VACADYEAR       IN NUMBER,  
    VSEMESTER       IN NUMBER  
)

## 1️⃣ ตรวจสอบสถานะการลงทะเบียน

ตรวจสอบค่า

|Status|Meaning|
|---|---|

|L|ลาพัก|
