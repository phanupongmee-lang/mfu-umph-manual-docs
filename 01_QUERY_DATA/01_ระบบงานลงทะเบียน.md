
## 1. ดึงข้อมูลนักศึกษาที่ได้รับอนุมัติแจ้งความประสงค์ แต่ไม่ได้ลงทะเบียนเรียน

**วัตถุประสงค์:** เพื่อดึงรายชื่อและตรวจสอบข้อมูลของนักศึกษาที่ได้รับการอนุมัติในขั้นตอนแจ้งความประสงค์แล้ว (`approve_flag = 'Y'`) แต่ไม่พบประวัติการลงทะเบียนเรียนในเทอมนั้นๆ (ไม่มีข้อมูลในตาราง `enrollsummary`) **ผู้ร้องขอ:** เจ้าหน้าที่ทะเบียน / เจ้าหน้าที่คณะ **ตารางที่เกี่ยวข้อง:** `avsreg.pre_reg_dtl`, `avsreg.pre_reg_hd`, `avsreg.studentmaster`, `avsreg.enrollsummary` (และตาราง Master data เช่น faculty, department, officer)

**SQL Scripts & Explanations:**

### 1.1 ดึงรายชื่อนักศึกษาและอาจารย์ที่ปรึกษา (ภาพรวมตามปี/เทอม)

ใช้สำหรับดึงรายชื่อนักศึกษาแบบละเอียด (มีชื่อ-นามสกุล, คณะ, สาขา และชื่ออาจารย์ที่ปรึกษา) โดยใช้เงื่อนไข `NOT IN` เพื่อกรองคนที่ลงทะเบียนใน `enrollsummary` ออกไป

SQL

```
SELECT s.studentcode, p.prefixname, s.studentname, s.studentsurname, 
       f.facultyname, de.departmentname, o.officername || ' ' || o.officersurname AS ที่ปรึกษา
FROM avsreg.pre_reg_dtl d 
INNER JOIN AVSREG.PRE_REG_HD h ON h.pre_reg_hd_id = d.pre_reg_hd_id 
INNER JOIN avsreg.studentmaster s ON s.studentid = h.studentid
INNER JOIN avsreg.prefix p ON p.prefixid = s.prefixid
INNER JOIN avsreg.faculty f ON f.facultyid = s.facultyid 
INNER JOIN avsreg.department de ON de.departmentid = s.departmentid
INNER JOIN avsreg.officer o ON o.officerid = s.officerid
WHERE d.approve_flag = 'Y' 
  AND h.acadyear = 2568 
  AND h.semester = 1 
  AND d.studentid NOT IN (
      SELECT e.studentid 
      FROM avsreg.enrollsummary e 
      WHERE e.acadyear = 2568 AND e.semester = 1
  )
GROUP BY s.studentcode, s.studentname, s.studentsurname, f.facultyname, 
         de.departmentname, o.officername, o.officersurname, p.prefixname;
```

### 1.2 ตรวจสอบการลงทะเบียนตกหล่นข้ามเทอม/ปี (Cross-check)

ใช้รีเช็คข้อมูลในตาราง `enrollsummary` อีกครั้ง (เช่น ในปี 2567) เพื่อหาความผิดปกติของข้อมูลกลุ่มที่ผ่านแจ้งความประสงค์แต่ไม่มีตัวตนในการลงทะเบียน

SQL

```
SELECT * FROM avsreg.enrollsummary e 
WHERE e.acadyear = 2567 
  AND e.studentid IN (
      SELECT s.studentid
      FROM avsreg.pre_reg_dtl d 
      INNER JOIN AVSREG.PRE_REG_HD h ON h.pre_reg_hd_id = d.pre_reg_hd_id 
      INNER JOIN avsreg.studentmaster s ON s.studentid = h.studentid
      WHERE d.approve_flag = 'Y' 
        AND h.acadyear = 2567 
        AND h.semester = 1 
        AND d.studentid NOT IN (
            SELECT e.studentid 
            FROM avsreg.enrollsummary e 
            WHERE e.acadyear = 2567
        )
  );
```

### 1.3 เจาะดูรายละเอียดการแจ้งความประสงค์รายบุคคล (Drill-down)

ใช้เมื่อต้องการตรวจสอบสถานะแบบเจาะจงรายบุคคล (เช่น รหัส 6431503006) ว่าข้อมูล Detail ในการแจ้งความประสงค์ของเด็กคนนี้มีอะไรบ้าง

SQL

```
SELECT d.*
FROM avsreg.pre_reg_dtl d 
INNER JOIN AVSREG.PRE_REG_HD h ON h.pre_reg_hd_id = d.pre_reg_hd_id 
INNER JOIN avsreg.studentmaster s ON s.studentid = h.studentid
WHERE d.approve_flag = 'Y' 
  AND h.acadyear = 2566 
  AND h.semester = 1 
  AND d.studentid = 6431503006; -- เปลี่ยนเป็นรหัสนักศึกษาที่ต้องการตรวจสอบ
```

#### 🟢 Script 2.3.1: ดึงข้อมูลตารางเรียนและรายชื่อ (แสดงช่วงเวลาเรียน Time Slot)

**สำหรับวิชา 1006002:**

SQL

```
select c.coursecode, c.coursenameeng, e.section, 
avsreg.timeslot(t.timeslotfrom-1) ||' - '|| avsreg.timeslot(t.timeslotto) as time, 
sm.studentcode, p.prefixname, sm.studentname, sm.studentsurname, 
p.prefixnameeng, sm.studentnameeng, sm.studentsurnameeng, sm.mfuemail, 
d.departmentid, d.departmentnameeng, f.facultyid, f.facultynameeng
from avsreg.enrollsummary e
left join avsreg.studentmaster sm on sm.studentid = e.studentid
left join avsreg.course c on c.courseid = e.courseid
left join avsreg.department d on d.departmentid = sm.departmentid and d.facultyid = sm.facultyid
left join avsreg.faculty f on f.facultyid = sm.facultyid
left join avsreg.prefix p on p.prefixid = sm.prefixid
left join avsreg.classtimetable t on t.classid = e.classid
where sm.studentstatus = 10
and sm.admitacadyear = 2568
and sm.admitsemester = 1
and c.coursecode = '1006002'
group by c.coursecode, c.coursenameeng, e.section, 
avsreg.timeslot(t.timeslotfrom-1) ||' - '|| avsreg.timeslot(t.timeslotto), 
sm.studentcode, p.prefixname, sm.studentname, sm.studentsurname, 
p.prefixnameeng, sm.studentnameeng, sm.studentsurnameeng, sm.mfuemail, 
d.departmentid, d.departmentnameeng, f.facultyid, f.facultynameeng
order by e.section, sm.studentcode;
```

## 3. คู่มือ: การดึงยอดจำนวนนักศึกษาแจ้งความประสงค์ (แยกตามชั้นปี)

งานส่วนนี้คือการดึงรายงาน **"สรุปยอดสถิติ (Count)"** ของนักศึกษาที่เข้ามาทำรายการในระบบแจ้งความประสงค์ลงทะเบียนเรียน โดยแบ่งกลุ่มตามชั้นปี (ปี 1, 2, 3 และ 4 ขึ้นไป) และแยกตามสถานะการทำรายการ ได้แก่:

2. จำนวนที่กดยืนยัน (`confirm_flag = 'Y'`)
    
3. จำนวนที่ได้รับการอนุมัติ (`approve_flag = 'Y'`)
    

**วัตถุประสงค์:** เพื่อส่งยอดสรุปให้เจ้าหน้าที่นำไปวิเคราะห์ภาพรวม หรือใช้ประเมินจำนวนการเปิดรายวิชา/Section

## 4. คู่มือ: การดึงยอดจำนวนนักศึกษาลงทะเบียน (แยกตามช่วงเวลา/วัน และเปรียบเทียบยอดอนุมัติ)

### 📌 4.1 Overview ของงาน

งานส่วนนี้คือการรัน Report เพื่อดู **"ความคืบหน้าการลงทะเบียนเรียน"** โดยเปรียบเทียบระหว่าง "จำนวนนักศึกษาที่ลงทะเบียนสำเร็จด้วยตัวเองในช่วงเวลาที่กำหนด" กับ "ยอดนักศึกษาที่ได้รับอนุมัติจากระบบแจ้งความประสงค์" โดยแบ่งข้อมูลแยกตามชั้นปี (ปี 1, 2, 3 และ 4 ขึ้นไป)

งานส่วนนี้คือการรัน Report เพื่อดึงข้อมูล **"นักศึกษาที่ทำเรื่องขอย้ายสาขาวิชาหรือย้ายคณะ"** โดยคิวรี่จะทำการเปรียบเทียบข้อมูลสังกัดเดิม (คณะ/สาขาเก่า และ รหัสนักศึกษาเก่า) กับสังกัดใหม่ (คณะ/สาขาใหม่ และ รหัสนักศึกษาใหม่ที่ได้รับหลังย้ายสาขา)

- **วัตถุประสงค์:** เพื่อให้เจ้าหน้าที่ใช้ตรวจสอบความถูกต้องของข้อมูลการย้ายสังกัด และดูรายชื่อนักศึกษาที่ย้ายเข้ามาในคณะเป้าหมายตามช่วงรหัสชั้นปีที่กำหนด
    

### ✅ 7.2 Checklist ก่อนเริ่มงาน (Pre-requisites)

- **โปรแกรม:** ติดตั้งโปรแกรม **PL/SQL Developer**
    
- **Access Rights:** สิทธิ์เข้าถึง Database Schema `AVSREG`
    
- **จุดที่ต้องปรับแก้ใน Code (สำคัญ 🚨):** ผู้รันคิวรี่จะต้องแก้ไขเงื่อนไข 2 จุดหลักในบรรทัด `WHERE` ให้ตรงกับความต้องการของเจ้าหน้าที่ ได้แก่:
    
    1. `newfacultyid` = รหัสคณะปลายทางที่นักศึกษาย้ายไป (เช่น 10)
        
    2. `SUBSTR(t.newstudentcode,0,2)` = รหัส 2 ตัวแรกของรหัสนักศึกษาใหม่ (มักจะหมายถึงปีการศึกษา เช่น `65` หมายถึงรหัส 65xxxx...)

## 8. คู่มือ: การติดตามนักศึกษาที่ตกหล่นรายวิชา (MFU333 - กรณีไม่ผ่าน และ ยังไม่เคยลงทะเบียน)

### 📌 8.1 Overview ของงาน

งานส่วนนี้คือการตรวจสอบและดึงรายชื่อนักศึกษาสถานะปกติ (Active) ที่มีปัญหาการตกหล่นในรายวิชาเป้าหมาย (ในที่นี้คือ `MFU333`) โดยแบ่งออกเป็น 2 กลุ่มเป้าหมายหลัก:

1. **กลุ่มที่ลงเรียนแล้วแต่ "ไม่ผ่าน" (ไม่ตก S):** ค้นหานักศึกษาที่เคยลงทะเบียนเรียน ได้เกรดอื่นๆ ที่ไม่ใช่ S หรือ W และปัจจุบันก็ยังไม่ได้กลับมาลงเรียนจนผ่าน
    
2. **กลุ่มที่ "ไม่เคยลงทะเบียนเรียนเลย":** ค้นหานักศึกษาระดับชั้นปริญญาตรี (`levelid = 3`) ที่ไม่เคยมีประวัติการลงทะเบียนในรายวิชานี้มาก่อนเลย
    

- **วัตถุประสงค์:** เพื่อส่งรายชื่อให้เจ้าหน้าที่นำไปติดตาม แจ้งเตือน หรือบังคับลงทะเบียนเรียนให้ครบถ้วนก่อนสำเร็จการศึกษา
    

### ✅ 8.2 Checklist ก่อนเริ่มงาน (Pre-requisites)

- **โปรแกรม:** ติดตั้งโปรแกรม **PL/SQL Developer**
    
- **Access Rights:** สิทธิ์เข้าถึง Database Schema `AVSREG`
    
- **จุดที่ต้องระวังในการแก้ Code (สำคัญ 🚨):** หากในอนาคตต้องการตรวจสอบรายวิชาอื่นที่ไม่ใช่ MFU333 ผู้ปฏิบัติงานต้องเข้าไปแก้รหัสวิชาใน Code (ซึ่งมีให้แก้หลายจุด ทั้งใน Main Query และ Subquery)
    

### ⚙️ 8.3 ขั้นตอนการทำงาน (Simplified Workflow)

1. เปิดโปรแกรม PL/SQL Developer และ Login เข้าสู่ระบบ
    
2. เลือกรัน Script ตามกลุ่มเป้าหมายที่ต้องการค้นหา (ตาม Code ด้านล่าง)
    
3. หากต้องการเปลี่ยนรายวิชา ให้ค้นหาคำว่า `'MFU333'` แล้วแทนที่ด้วยรหัสวิชาใหม่
    
4. กด Run (Execute) และ Export ผลลัพธ์เป็นไฟล์ Excel ส่งให้เจ้าหน้าที่ที่เกี่ยวข้อง
    

#### 🟢 Script 8.3.1: กลุ่มที่ 1 - เคยลงทะเบียนแล้วแต่ "ไม่ผ่าน" (เกรดไม่ใช่ S)

SQL

```
select c.coursecode, c.coursenameeng, s.studentcode, v.PREFIXNAMEENG, s.studentnameeng, s.studentsurnameeng, 
v.DEPARTMENTID, v.departmentnameeng, v.FACULTYID, v.FACULTYNAMEENG, s.studentstatus, b.bytedes, t.grade
from AVSREG.ENROLLSUMMARY t  
inner join avsreg.course c on c.courseid = t.courseid
inner join avsreg.studentmaster s on s.studentid = t.studentid
inner join avsreg.sysbytedes b on b.bytecode = s.studentstatus and b.tablename = 'STUDENTSTATUS' and b.columnname ='STUDENTSTATUS'
inner join avsreg.viewstudentinfo v on v.STUDENTID = s.studentid

/* เงื่อนไขหลัก: สถานะ 10, เกรดไม่ใช่ S และ W, และไม่ใช่คนที่เคยสอบผ่านไปแล้ว (Subquery ด้านล่าง) */
 where /*t.acadyear > 2561  and*/ s.studentstatus between 10 and 10  
 and t.grade not in  ('S','W')  
 and c.coursecode IN ( 'MFU333' ) /*and s.facultyid = 22*/ 
 and v.STUDENTID not in (
    select t.studentid from AVSREG.ENROLLSUMMARY t  
    inner join avsreg.course c on c.courseid = t.courseid
    inner join avsreg.studentmaster s on s.studentid = t.studentid
    inner join avsreg.sysbytedes b on b.bytecode = s.studentstatus and b.tablename = 'STUDENTSTATUS' and b.columnname ='STUDENTSTATUS'
    inner join avsreg.viewstudentinfo v on v.STUDENTID = s.studentid
    where s.studentstatus between 10 and 10  --and s.facultyid = 22
    and c.coursecode IN ( 'MFU333') and t.grade = 'S'
) 
 group by c.coursecode, c.coursenameeng, s.studentcode, v.PREFIXNAMEENG, s.studentnameeng, s.studentsurnameeng, 
v.DEPARTMENTID, v.departmentnameeng, v.FACULTYID, v.FACULTYNAMEENG, s.studentstatus, b.bytedes, t.grade
 order by c.coursecode;
```

## 9. คู่มือ: การดึงสถิติจำนวนอนุมัติรับและยืนยันรับ (Request & Confirm Seats)

2. **การดึงสถิติภาพรวม:** ดูยอดรวมที่นั่งทั้งหมด (Total Seat), ยอดคนลงทะเบียนจริง (Enroll Seat) และยอดคนที่ได้รับการยืนยัน (Confirm Qty) ในแต่ละกลุ่มเรียน (Section) ของระดับปริญญาตรี (`levelid = 3`)
    

### ✅ 9.2 Checklist ก่อนเริ่มงาน (Pre-requisites)

- **Access Rights:** สิทธิ์เข้าถึง Database Schema `AVSREG`
    
- **จุดที่ต้องปรับแก้ใน Code (สำคัญ 🚨):** ก่อนรันคิวรี่ **ต้องเปลี่ยนปีการศึกษา (`acadyear`) และภาคเรียน (`semester`)** ในโค้ดให้เป็นปัจจุบัน (มีจุดให้แก้ในทั้ง 2 Scripts) และหากต้องการดูเฉพาะเจาะจงบางวิชา สามารถลบเครื่องหมาย Comment (`--`) หน้า `co.coursecode` แล้วใส่รหัสวิชาที่ต้องการได้เลย

## 10. คู่มือ: การดึงข้อมูลนักศึกษาต่างชาติที่ลงทะเบียนเรียน (เช็คยอดค่าเทอม/ทุน/ผ่อนผัน)

### 📌 10.1 Overview ของงาน

งานส่วนนี้คือการรัน Report เพื่อดูข้อมูล **"นักศึกษาต่างชาติ"** ที่ลงทะเบียนเรียนในเทอมนั้นๆ โดยระบบจะแสดงรายชื่อ (ภาษาอังกฤษ), คณะ/สาขา, ยอดค่าธรรมเนียมการศึกษา (`amount`), ยอดค้างชำระ (`balance`) พร้อมทั้งมีคอลัมน์พิเศษที่เช็คสถานะว่านักศึกษาคนนั้น:

1. **ได้รับทุนการศึกษาหรือไม่?** (ถ้ามีจะแสดงคำว่า 'ทุน')
    
2. **มีการทำเรื่องผ่อนผันค่าเทอมหรือไม่?** (ถ้ามีจะแสดงคำว่า 'ผ่อนผัน')
    

- **เงื่อนไขสำคัญที่ใช้คัดกรอง:** คิวรี่นี้ใช้วิธีเช็คจาก `v.Check_Citizenid = 0` เพื่อระบุตัวตนว่าเป็นนักศึกษาชาวต่างชาติ (ไม่มีเลขบัตรประชาชนไทย)
    

### ✅ 10.2 Checklist ก่อนเริ่มงาน (Pre-requisites)

- **โปรแกรม:** ติดตั้งโปรแกรม **PL/SQL Developer**

## 14. คู่มือ: การตรวจสอบรายชื่อนักศึกษาที่สมัครเข้าเรียนใหม่ (Re-ID / Re-Entry)

งานส่วนนี้คือการรัน Report เพื่อค้นหา **"นักศึกษาหน้าเก่าในร่างใหม่"** หรือกลุ่มนักศึกษาที่เคยมีรหัสนักศึกษาเดิมในระบบมาก่อน แล้วกลับมาสมัครเข้าเรียนใหม่ในปีการศึกษาปัจจุบัน โดยคิวรี่จะใช้ **เลขที่บัตรประชาชน (Citizen ID)** เป็นตัวเชื่อมข้อมูลระหว่างประวัติเก่าและประวัติใหม่

- **วัตถุประสงค์:** เพื่อให้เจ้าหน้าที่ฝ่ายทะเบียนตรวจสอบประวัตินักศึกษาเดิม สำหรับใช้ประกอบการพิจารณาโอนคะแนนสะสม (Credit Transfer) หรือตรวจสอบความซ้ำซ้อนของข้อมูลในระบบ
    

### ✅ 14.2 Checklist ก่อนเริ่มงาน (Pre-requisites)

- **โปรแกรม:** ติดตั้งโปรแกรม **PL/SQL Developer**
    
- **Access Rights:** สิทธิ์เข้าถึง Database Schema `AVSREG`
    
- **จุดที่ต้องแก้ใน Code (สำคัญมาก 🚨):** คิวรี่นี้มีการระบุปีการศึกษาที่นักศึกษา "เข้าใหม่" ไว้หลายจุด ในตัวอย่างคือปี **2565** หากเป็นปีอื่นต้องไล่แก้ให้ครบทุกจุด ได้แก่:
    
    1. ใน Subquery แรก: `sbb.admitacadyear <> 2565` (เพื่อระบุว่า ID เก่าต้องไม่ใช่ปีนี้)
        
    2. ใน Inner Join: `sss.admitacadyear = 2565` (เพื่อระบุ ID ใหม่ของปีนี้)
        
    3. ในเงื่อนไข Applicant: `t.acadyear = 2565` (เพื่อเช็คข้อมูลการสมัคร)
        

### ⚙️ 14.3 ขั้นตอนการทำงาน (Simplified Workflow)

1. เปิดโปรแกรม PL/SQL Developer และ Login เข้าสู่ระบบฐานข้อมูล
    
2. คัดลอก Code SQL ด้านล่างนี้ไปวาง
    
3. ตรวจสอบปีการศึกษา (ตัวเลข 2565) และแก้ไขให้เป็นปีที่ต้องการตรวจสอบ
    
4. กด Run (Execute) และ Export ผลลัพธ์เป็นไฟล์ Excel ซึ่งจะแสดงทั้ง **รหัสเก่า และ รหัสใหม่** คู่กัน พร้อมสำนักวิชาที่เคยเรียนและที่เรียนอยู่ในปัจจุบัน
    

#### 🟢 Script 14.3.1: ดึงรายชื่อนักศึกษา Re-ID (รหัสเก่า vs รหัสใหม่)

SQL

```
select 
   sss.studentid as รหัสนักศึกษาใหม่,
   vi.studentcode as รหัสนักศึกษาเก่า,
   vi.PREFIXNAME as คำนำหน้า,
   vi.STUDENTNAME as ชื่อ,
   vi.STUDENTSURNAME as นามสกุล,
   vi.FACULTYNAME as สำนักวิชาเก่า,
   vi.DEPARTMENTNAME as สาขาวิชาเก่า,
   vii.FACULTYNAME as สำนักวิชาใหม่,
   vii.DEPARTMENTNAME as สาขาวิชาใหม่
from avsreg.viewstudentinfo vi
inner join (
    /* หาประวัติเก่าที่ไม่ได้เข้าในปีปัจจุบัน */
    select sbb.studentid, sbb.citizenid
    from avsreg.studentbio sbb
    where sbb.admitacadyear <> 2565   
    group by sbb.studentid, sbb.citizenid
) s on s.studentid = vi.studentid

/* เชื่อมข้อมูลด้วยบัตรประชาชนเพื่อหารหัสใหม่ในปีปัจจุบัน */
inner join avsreg.studentbio sss on vi.CITIZENID = sss.citizenid and sss.admitacadyear = 2565
inner join avsreg.viewstudentinfo vii on vii.STUDENTID = sss.studentid

where vii.STUDENTSTATUS between 10 and 18 
and sss.studentid in (
    select s.studentid from avsreg.applicant t 
    inner join avsreg.studentbio s on s.applicantid = t.applicantid
    inner join avsreg.entrygroup e on e.entrygroupid = t.entrygroupid
    where t.acadyear = 2565
)
order by sss.studentid;
```

## 16. คู่มือ: การดึงสถิติการลงทะเบียนแยกตามหมวดรายวิชา (Course Category)

### 📌 16.1 Overview ของงาน

งานส่วนนี้คือการรัน Report เพื่อวิเคราะห์ **"ปริมาณการลงทะเบียนเรียนของนักศึกษาในหมวดวิชาต่างๆ ตามที่ระบุไว้ในโครงสร้างหลักสูตร"** โดยคิวรี่จะทำการตรวจสอบว่า นักศึกษาในสาขาวิชา/คณะที่กำหนด (สังกัด `s.facultyid`) ไปลงทะเบียนในรายวิชาที่จัดอยู่ในหมวดหมู่เฉพาะ (ระบุโดย `conditionid`) มากน้อยเพียงใด

- **วัตถุประสงค์:** เพื่อใช้ตรวจสอบภาระงานสอนรายหมวดวิชา, ตรวจสอบที่นั่งคงเหลือในแต่ละกลุ่มเรียน (Section), และวิเคราะห์ความต้องการเรียนในหมวดวิชาเลือกหรือวิชาบังคับตามโครงสร้างหลักสูตร
    

### ✅ 16.2 Checklist ก่อนเริ่มงาน (Pre-requisites)

- **โปรแกรม:** ติดตั้งโปรแกรม **PL/SQL Developer**
    
- **Access Rights:** สิทธิ์เข้าถึง Database Schema `AVSREG`
    
- **ข้อมูลที่ต้องเตรียมแก้ไข (สำคัญ 🚨):** คิวรี่นี้มีการคัดกรองที่ซับซ้อนใน 3 จุดหลัก:
    
    1. **ปี/เทอม:** `t.acadyear = 2563` และ `t.semester = 2`
        
    2. **รหัสคณะของนักศึกษา:** `s.facultyid = 12`
        
    3. **หมวดวิชา (Condition ID):** ในส่วนของ Subquery จะมีการระบุรหัสหมวดวิชา เช่น `'1.1','1.2','1.4'` ซึ่งต้องปรับเปลี่ยนตามคู่มือรหัสหมวดวิชาของมหาวิทยาลัย
        

### ⚙️ 16.3 ขั้นตอนการทำงาน (Simplified Workflow)

1. เปิดโปรแกรม PL/SQL Developer และ Login เข้าสู่ระบบ
    
2. วาง Code SQL ด้านล่างนี้
    
3. **ตรวจสอบรหัสคณะ:** เปลี่ยน `s.facultyid = 12` เป็นรหัสคณะที่ต้องการดูข้อมูลนักศึกษา
    
4. **ตรวจสอบหมวดวิชา:** แก้ไขค่าใน `t.conditionid IN (...)` ให้ตรงกับหมวดวิชาที่ฝ่ายวิชาการต้องการตรวจสอบ
    
5. กด Run (Execute) และ Export ผลลัพธ์เพื่อดูจำนวนรับ (Total), จำนวนลง (Enroll) และจำนวนที่นั่งคงเหลือแยกตามรายวิชาและกลุ่มเรียน
    

#### 🟢 Script 16.3.1: ดึงสถิติการลงทะเบียนตามหมวดรายวิชาในหลักสูตร

SQL

```
select 
    c.coursecode, 
    c.coursenameeng, 
    t.section, 
    o.officercode, 
    o.officernameeng, 
    o.officersurnameeng, 
    c.courseunit,
    t.totalsumseat as จำนวนรับ, 
    t.enrollseat as จำนวนลง, 
    t.totalsumseat - t.enrollseat as จำนวนที่นั่งคงเหลือ,
    s.departmentid, 
    count(s.studentid) as จำนวนนักศึกษา
from AVSREG.CLASS t
inner join avsreg.classinstructor ci on ci.classid = t.classid 
inner join avsreg.officer o on o.officerid = ci.officerid
inner join avsreg.enrollsummary e on e.classid = t.classid 
inner join avsreg.studentmaster s on e.studentid = s.studentid
inner join avsreg.course c on c.courseid = t.courseid 
inner join avsreg.faculty f on f.facultyid = s.facultyid

/* 🚨 จุดที่ต้องแก้ไข: รหัสคณะนักศึกษา, ลำดับอาจารย์ (ไม่เอา 99,90), ปี/เทอม */
where s.facultyid = 12 
and ci.sequence not in (99,90)
and t.acadyear = 2563 
and t.semester = 2 

/* 🚨 จุดสำคัญ: การคัดกรองตามหมวดรายวิชาในหลักสูตร (Condition ID) */
and c.courseid in (
    select co.courseid 
    from AVSREG.COURSEINPROGRAM t 
    inner join avsreg.course co on co.courseid = t.courseid 
    where t.programid = s.programid 
    and co.facultyid = 10 -- ระบุคณะเจ้าของวิชา
    and t.conditionid in ('1.1','1.2','1.4') -- ระบุรหัสหมวดวิชาที่ต้องการ
)

group by 
    c.coursecode, c.coursenameeng, t.section, o.officercode, o.officernameeng, 
    o.officersurnameeng, c.courseunit, t.totalsumseat, t.enrollseat, s.departmentid
order by c.coursecode, t.section;
```

## 17. คู่มือ: การดึงสถิติจำนวนผู้ลงทะเบียนรายวิชา เปรียบเทียบย้อนหลัง (Enrollment Trend Report)

### 📌 17.1 Overview ของงาน

งานส่วนนี้คือการสร้างรายงานสรุป **"จำนวนนักศึกษาที่ลงทะเบียนเรียนในแต่ละรายวิชา"** โดยแสดงผลแบบตารางเปรียบเทียบแยกตามปีการศึกษาและภาคเรียน (Pivot Table) ย้อนหลังตั้งแต่ปี 2559 ถึงปัจจุบัน

- **วัตถุประสงค์:** เพื่อวิเคราะห์ความนิยมของรายวิชา, วางแผนการจัดการทรัพยากรผู้สอน, และคาดการณ์จำนวนการรับนักศึกษาในเทอมถัดไปจากการดูสถิติย้อนหลัง
    

### ✅ 17.2 Checklist ก่อนเริ่มงาน (Pre-requisites)

- **โปรแกรม:** ติดตั้งโปรแกรม **PL/SQL Developer**
    
- **Access Rights:** สิทธิ์เข้าถึง Database Schema `AVSREG`
    
- **ข้อมูลที่ต้องเตรียมแก้ไข (สำคัญ 🚨):**
    
    1. **รายชื่อวิชา:** แก้ไขในส่วน `c.coursecode IN (...)` โดยระบุรหัสวิชาที่ต้องการเปรียบเทียบ
        
    2. **ช่วงปีการศึกษา:** หากต้องการข้อมูลปีที่ใหม่กว่า 2563 ผู้ปฏิบัติงานต้องเพิ่มบรรทัด `sum(case when t.acadyear = 25xx ...)` เพิ่มเติมในส่วน SELECT เพื่อขยายฐานข้อมูลในรายงาน
        

### ⚙️ 17.3 ขั้นตอนการทำงาน (Simplified Workflow)

1. เปิดโปรแกรม PL/SQL Developer และ Login
    
2. วาง Code SQL ด้านล่างนี้
    
3. **กำหนดรายวิชา:** แก้ไขรหัสวิชาในวงเล็บ `IN (...)` ให้เป็นวิชาที่ต้องการดู Trend
    
4. **ตรวจสอบสถานะวิชา:** คิวรี่นี้ตั้งค่า `c.coursestatus = 'O'` (Open) เพื่อดูเฉพาะวิชาที่ยังเปิดสอนอยู่ปัจจุบัน
    
5. กด Run (Execute) และ Copy ข้อมูลทั้งหมดไปวางใน **Excel** เพื่อนำไปสร้างกราฟหรือทำรายงานสรุปเสนอผู้บริหาร
    

#### 🟢 Script 17.3.1: รายงานสถิติจำนวนผู้ลงทะเบียนแยกตามรายวิชาและปีการศึกษา (Trend)

SQL

```
select 
    b.bytedes as สถานะวิชา, 
    s.bytedes as กลุ่มวิชา,  
    c.coursecode as รหัสวิชา,
    f.facultynameeng as สำนักวิชา,
    c.coursenameeng as ชื่อวิชา, 
    c.courseunit as หน่วยกิต, 
    
    /* สถิติจำนวนคนลงทะเบียนแยกตามปีและเทอม (Pivot) */
    sum (case when t.acadyear = 2559 and t.semester = 1 then 1 else 0 end) as Y59_1,
    sum (case when t.acadyear = 2559 and t.semester = 2 then 1 else 0 end) as Y59_2,
    sum (case when t.acadyear = 2560 and t.semester = 1 then 1 else 0 end) as Y60_1,
    sum (case when t.acadyear = 2560 and t.semester = 2 then 1 else 0 end) as Y60_2,
    sum (case when t.acadyear = 2561 and t.semester = 1 then 1 else 0 end) as Y61_1,
    sum (case when t.acadyear = 2561 and t.semester = 2 then 1 else 0 end) as Y61_2,
    sum (case when t.acadyear = 2562 and t.semester = 1 then 1 else 0 end) as Y62_1,
    sum (case when t.acadyear = 2562 and t.semester = 2 then 1 else 0 end) as Y62_2,
    sum (case when t.acadyear = 2563 and t.semester = 1 then 1 else 0 end) as Y63_1,
    sum (case when t.acadyear = 2563 and t.semester = 2 then 1 else 0 end) as Y63_2  
      
from AVSREG.enrollsummary t 
inner join avsreg.course c on c.courseid = t.courseid
inner join avsreg.faculty f on f.facultyid = c.facultyid
inner join avsreg.sysbytedes b on b.tablename = 'COURSE' and b.columnname = 'COURSESTATUS' and b.bytecode = c.coursestatus
inner join avsreg.sysbytedes s on s.tablename = 'COURSE' and s.columnname = 'COURSEGROUP' and s.bytecode = c.coursegroup

/* 🚨 จุดที่ต้องแก้ไข: ใส่รหัสวิชาที่ต้องการตรวจสอบ */
where c.coursecode in ('1008111', '1008112', '1008113', '1008114', '1008321')
and c.coursestatus = 'O' -- เฉพาะวิชาที่สถานะปกติ (Open)

group by b.bytedes, s.bytedes, c.coursecode, c.coursenameeng, c.courseid, f.facultyid, f.facultynameeng, c.courseunit
order by f.facultyid, c.coursecode;
```
