## 1. คู่มือ: การตรวจสอบตารางสอบซ้ำซ้อน (Exam Conflict Detection)

### 📌 1.1 Overview ของงาน

งานส่วนนี้คือการตรวจสอบ **"ความซ้ำซ้อนของวันและเวลาสอบ"** ของนักศึกษาแต่ละรายบุคคล โดยระบบจะทำการสแกนรายวิชาที่นักศึกษาลงทะเบียนเรียนทั้งหมด แล้วตรวจสอบว่ามีรายวิชาใดบ้างที่มี **วันสอบเดียวกัน (Exam Date)** และ **ช่วงเวลาเริ่มสอบเดียวกัน (Exam Time From)** * **วัตถุประสงค์:** เพื่อตรวจหา Conflict ก่อนที่จะมีการประกาศตารางสอบหรือจัดห้องสอบจริง หากพบรายชื่อนักศึกษาใน Report นี้ เจ้าหน้าที่ต้องดำเนินการแจ้งคณะเพื่อปรับเปลี่ยนวันเวลาสอบ หรือจัดสอบแยกกรณีพิเศษให้นักศึกษา

### ✅ 1.2 Checklist ก่อนเริ่มงาน (Pre-requisites)

- **โปรแกรม:** ติดตั้งโปรแกรม **PL/SQL Developer**
    
- **Access Rights:** สิทธิ์เข้าถึง Database Schema `AVSREG`
    
- **ข้อมูลที่ต้องเตรียมแก้ไข (สำคัญ 🚨):**
    
    1. **ประเภทการสอบ (`examcode`):** ใน Code ตั้งไว้ที่ `'M'` (Midterm) หากต้องการตรวจสอบสอบปลายภาคให้เปลี่ยนเป็น `'F'` (Final)
        
    2. **ปีการศึกษา/ภาคเรียน:** แก้ไข `e.acadyear = 2567` และ `e.semester = 2` ให้เป็นปัจจุบัน
        

### ⚙️ 1.3 ขั้นตอนการทำงาน (Simplified Workflow)

1. เปิดโปรแกรม PL/SQL Developer และ Login
    
2. วาง Code SQL ในส่วนของ Script 1.3.1
    
3. **กำหนดเงื่อนไข:** ตรวจสอบรหัสปีการศึกษา ภาคเรียน และประเภทการสอบ (กลางภาค/ปลายภาค)
    
4. กด Run (Execute)
    
    - **หากผลลัพธ์ว่างเปล่า (No rows found):** แสดงว่าไม่มีนักศึกษาคนใดมีตารางสอบซ้ำซ้อน (ข้อมูลปกติ)
        
    - **หากพบรายชื่อ:** ให้ดูที่คอลัมน์ `CNT` (หากมากกว่า 1 แสดงว่าวิชาในวัน/เวลานั้นชนกัน) ให้ส่งรายชื่อนักศึกษาและวันที่ชนกันให้งานจัดสอบตรวจสอบทันที
        

#### 🟢 Script 1.3.1: ตรวจสอบนักศึกษาที่มีตารางสอบกลางภาค/ปลายภาคซ้ำซ้อน

SQL

```
select sett.*
 from (
    /* Subquery: จัดกลุ่มข้อมูลตามรหัสนักศึกษา วันที่สอบ และเวลาเริ่มสอบ */
    select count(distinct c.classid) cnt, sm.studentcode, c.examdate, to_char(c.examtimefrom,'hh24:mi:ss') as StartTime
    from avsreg.enrollsummary e
    left join avsreg.classexam c on c.classid = e.classid 
    left join avsreg.studentmaster sm on sm.studentid = e.studentid
    
    where 1=1
    /* 🚨 จุดที่ต้องแก้ไข: ประเภทการสอบ (M=Midterm, F=Final) และ ปี/เทอม */
    and c.examcode = 'M' 
    and e.acadyear = 2567
    and e.semester = 2
    and c.examdate is not null

    group by sm.studentcode, c.examdate, to_char(c.examtimefrom,'hh24:mi:ss')
    order by sm.studentcode, c.examdate, to_char(c.examtimefrom,'hh24:mi:ss')
 ) sett
where 1=1
/* เงื่อนไขสำคัญ: ดึงเฉพาะรายการที่นับจำนวนกลุ่มเรียน (Class) ได้มากกว่า 1 ในเวลาเดียวกัน */
and sett.cnt > 1;
```

## 2. คู่มือ: การตรวจสอบความครบถ้วนของข้อมูลเมื่อมีการย้ายสาขาวิชา (Major Change Data Audit)

## 3. คู่มือ: การตรวจสอบรายละเอียดการบันทึกเกรดรายวิชา (Grade Entry Audit)

### 📌 3.1 Overview ของงาน

ชุดคำสั่งนี้ใช้สำหรับ **"ตรวจสอบสถานะการให้เกรด"** ของนักศึกษาในแต่ละรายวิชา โดยจะแสดงข้อมูลตั้งแต่การบันทึกคะแนนดิบหรือเกรดในแต่ละลำดับ (`gradeentry1`, `2`, `3`) ไปจนถึงเกรดสุทธิที่ระบบสรุปผล (`grade`) พร้อมทั้งแสดงชื่ออาจารย์ผู้สอนประจำกลุ่มเรียน (Section) นั้นๆ

- **วัตถุประสงค์:** เพื่อติดตามความคืบหน้าในการส่งเกรดของอาจารย์, ตรวจสอบความผิดปกติของข้อมูลเกรดในแต่ละ Entry และใช้ยืนยันสถานภาพนักศึกษาควบคู่ไปกับผลการเรียนในเทอมนั้นๆ
    

### ✅ 3.2 Checklist ก่อนเริ่มงาน (Pre-requisites)

- **โปรแกรม:** **PL/SQL Developer**
    
- **Access Rights:** สิทธิ์เข้าถึงข้อมูลผลการเรียน (`enrollsummary`) และข้อมูลบุคคลากร (`officer`)
    
- **จุดที่ต้องแก้ใน Code (สำคัญ 🚨):** 1. **ปี/เทอม:** แก้ไข `e.acadyear = 2567` และ `e.semester = 1` ให้เป็นปัจจุบัน 2. **รหัสวิชา:** ระบุรหัสที่ต้องการตรวจสอบใน `c.coursecode in ('2201223')` หรือหากต้องการดูทั้งสำนักวิชา ให้เปิด Comment ตรง `c.facultyid` แทน
    

### ⚙️ 3.3 ขั้นตอนการทำงาน (Step-by-Step)

1. เปิดโปรแกรม PL/SQL Developer และ Login
    
2. คัดลอก Code SQL ใน Script 3.3.1 ไปวาง
    
3. **กำหนดเงื่อนไขการค้นหา:**
    
    - ตรวจสอบว่าปีการศึกษาและภาคเรียนถูกต้องหรือไม่
        
    - ระบุรหัสวิชาที่ต้องการตรวจสอบ
        
4. กด Run (Execute) และตรวจสอบคอลัมน์ `gradeentry` ต่างๆ เพื่อดูว่าเกรดถูกบันทึกเข้ามาครบทุกขั้นตอนแล้วหรือไม่
    

#### 🟢 Script 3.3.1: ตรวจสอบรายละเอียดเกรดรายวิชาและผู้สอน

SQL

```
select 
    e.acadyear, e.semester, 
    c.coursecode, c.coursenameeng, 
    e.gradeentry1, e.gradeentry2, e.gradeentry3, e.grade,
    o.officercode ||' : '|| p.prefixname || o.officername ||' '|| o.officersurname as officer, 
    e.section,
    s.studentid, st.studentstatus, s.studentname, s.studentsurname 
from avsreg.studentmaster s
inner join avsreg.enrollsummary e on e.studentid = s.studentid
inner join avsreg.studentstatus st on st.studentid = s.studentid and st.acadyear = e.acadyear and st.semester = e.semester
inner join avsreg.course c on c.courseid = e.courseid
inner join avsreg.class cl on cl.classid = e.classid
inner join avsreg.classinstructor ci on ci.classid = cl.classid
inner join avsreg.officer o on o.officerid = ci.officerid
inner join avsreg.prefix p on p.prefixid = o.prefixid 

/* 🚨 จุดที่ต้องแก้ไข: ปี/เทอม และ รหัสวิชาที่ต้องการตรวจสอบ */
where e.acadyear = 2567 
and e.semester = 1
and c.coursecode in ('2201223') -- ใส่รหัสวิชาที่นี่

order by c.coursecode;
```

## 4. คู่มือ: การวิเคราะห์สถิติ GPAX แยกตามช่วงคะแนนและวิธีรับเข้า (GPAX Distribution by Entry Type)

### 📌 4.1 Overview ของงาน

ชุดคำสั่งนี้จะทำการประมวลผลข้อมูล **"ช่วงคะแนนเฉลี่ยสะสม (GPAX Range)"** ของนักศึกษาชั้นปีที่ 1 (หรือชั้นปีที่กำหนด) โดยแบ่งกลุ่มตาม **"วิธีรับเข้าเรียน"** (Entry Type)

- **วัตถุประสงค์:** เพื่อรายงานสถิติต่อผู้บริหารหรือหน่วยงานรับเข้าศึกษา (Admissions) ให้เห็นภาพรวมว่านักศึกษาในแต่ละสาขาวิชาส่วนใหญ่มีผลการเรียนเกาะกลุ่มอยู่ที่ช่วงคะแนนใด และช่องทางการรับเข้าใดที่นักศึกษามีศักยภาพการเรียนสูงสุด/ต่ำสุด
    

### ✅ 4.2 Checklist ก่อนเริ่มงาน (Pre-requisites)

- **โปรแกรม:** **PL/SQL Developer**
    
- **Access Rights:** สิทธิ์เข้าถึงตาราง `studentstatus` (สำหรับ GPAX) และ `studentbio` (สำหรับวิธีรับเข้า)
    
- **จุดที่ต้องแก้ใน Code (สำคัญ 🚨):** 1. **ปี/เทอม:** แก้ไข `ss.acadyear = 2565` และ `ss.semester = 1` 2. **รุ่นนักศึกษา:** แก้ไข `sm.admitacadyear = 2565` เพื่อระบุรุ่นนักศึกษาที่ต้องการวิเคราะห์ 3. **หน่วยงาน:** แก้ไข `sm.facultyid` (รหัสสำนักวิชา) และ `sm.departmentid` (รหัสสาขาวิชา) ตามที่ได้รับมอบหมาย
    

### ⚙️ 1.3 ขั้นตอนการทำงาน (Step-by-Step)

1. เปิดโปรแกรม PL/SQL Developer และ Login
    
2. วาง Code SQL ในส่วนของ Script 4.3.1
    
3. **ตรวจสอบเงื่อนไขการกรอง:** มั่นใจว่าได้ตั้งค่า `LEVELID = 3` (ระดับ ป.ตรี) และช่วงสถานภาพ `10-18` (สถานะปกติ)
    
4. กด Run (Execute) ผลลัพธ์จะแสดงผลเป็นตารางสรุปจำนวนคนแยกตามช่วงคะแนน (0.00-1.49, 1.50-1.99, ไปจนถึง 4.00)
    

#### 🟢 Script 4.3.1: สรุปจำนวนนักศึกษาแยกตามช่วง GPAX และช่องทางการรับเข้า

SQL

```
select 
    sb.entrytype ||' : '|| bd.bytedes as วิธีรับเข้า,
    d.departmentid || ' : ' || d.departmentname as สาขาวิชา,
    
    /* ประมวลผลช่วงคะแนน GPAX ด้วย CASE WHEN */
    SUM (case when ss.gpax between 0.00 and 4.00 then 1 else 0 end) as TOTAL_STUDENTS,
    SUM (case when ss.gpax between 0.00 and 1.49 then 1 else 0 end) as GPAX_UNDER_1_49, 
    SUM (case when ss.gpax between 1.50 and 1.99 then 1 else 0 end) as GPAX_1_50_1_99, 
    SUM (case when ss.gpax between 2.00 and 2.49 then 1 else 0 end) as GPAX_2_00_2_49,
    SUM (case when ss.gpax between 2.50 and 2.99 then 1 else 0 end) as GPAX_2_50_2_99, 
    SUM (case when ss.gpax between 3.00 and 3.49 then 1 else 0 end) as GPAX_3_00_3_49, 
    SUM (case when ss.gpax between 3.50 and 4.00 then 1 else 0 end) as GPAX_3_50_4_00
                            
from avsreg.studentstatus ss 
left join avsreg.studentmaster sm on ss.studentid = sm.studentid
left join avsreg.studentbio sb on sm.studentid = sb.studentid
left join avsreg.sysbytedes bd on sb.entrytype = bd.bytecode 
    and bd.tablename = 'STUDENTBIO' and bd.columnname = 'ENTRYTYPE'
left join avsreg.department d on sm.facultyid = d.facultyid and sm.departmentid = d.departmentid 

/* 🚨 แก้ไขเงื่อนไข ปีการศึกษา ภาคเรียน และหน่วยงานที่นี่ */
where ss.acadyear = 2565 
and ss.semester = 1 
and sm.studentstatus between 10 and 18 
and SM.LEVELID = 3 
and sm.admitacadyear = 2565  
and sm.facultyid = 17 
and sm.departmentid = 31701

group by ss.acadyear, ss.semester, d.departmentid, d.departmentname, sb.entrytype, bd.bytedes
order by d.departmentid, sb.entrytype;
```

## 5. คู่มือ: การตรวจสอบผลการเรียนรายวิชา MFU333 (Checking Non-Passing Grades)

ชุดคำสั่งนี้ใช้สำหรับ **"คัดกรองนักศึกษาที่ยังไม่ผ่านเงื่อนไขรายวิชา MFU333"** โดยระบบจะทำการดึงรายชื่อนักศึกษาชั้นปีที่กำหนด (ในที่นี้คือรหัส 62 ขึ้นไป) ที่ลงทะเบียนเรียนวิชานี้แล้ว แต่ผลการเรียนล่าสุด **"ยังไม่ใช่ S"** (และไม่ใช่ W)

### ⚙️ 5.3 ขั้นตอนการทำงาน (Step-by-Step)

1. เปิดโปรแกรม PL/SQL Developer และ Login
    
2. คัดลอก Code SQL ใน Script 5.3.1 ไปวาง
    
3. **ตรวจสอบ Subquery:** มั่นใจว่าในส่วน `(select MAX(t.grade)...)` มีการเช็คเกรดที่ไม่ใช่ `'S'` และไม่เอา `'W'` เพื่อให้ได้รายชื่อคนที่ต้องลงเรียนซ้ำหรือยังไม่ผ่านจริง ๆ
    
4. กด Run (Execute) และส่งรายชื่อให้เจ้าหน้าที่ฝ่ายทะเบียนที่ดูแลกลุ่มวิชาศึกษาทั่วไป (GE) เพื่อแจ้งเตือนนักศึกษา
    

#### 🟢 Script 5.3.1: ตรวจสอบนักศึกษาที่ยังไม่ผ่านวิชา MFU333

SQL

```
select 
    s.studentcode, v.PREFIXNAME, s.studentname, s.studentsurname, 
    v.FACULTYNAME, v.departmentname, s.studentyear, v.SEX, 
    b.bytedes as status_name, s.studentstatus, e.grade
from avsreg.viewstudentinfo v
inner join avsreg.enrollsummary e on e.studentid = v.STUDENTID
inner join avsreg.course c on c.courseid = e.courseid  
inner join avsreg.studentmaster s on s.studentid = v.studentid
inner join avsreg.sysbytedes b on b.bytecode = s.studentstatus 
    and b.tablename = 'STUDENTSTATUS' and b.columnname ='STUDENTSTATUS'

/* 🚨 เงื่อนไขการกรอง: รหัส 62 ขึ้นไป, ป.ตรี, วิชา MFU333 */
where s.studentstatus between 10 and 14 
and s.admitacadyear > 2561 
and s.levelid = 3
and c.coursecode IN ('MFU333') 

/* 🚨 Subquery สำคัญ: ตรวจสอบว่าเกรดล่าสุดไม่ใช่ 'S' และไม่ใช่ 'W' */
and (select MAX(t.grade) from AVSREG.ENROLLSUMMARY t  
     inner join avsreg.course c on c.courseid = t.courseid 
     where t.studentid = s.studentid and c.coursecode IN ('MFU333') 
     and t.grade <> 'W' ) <> 'S' 
and e.grade <> 'W'

group by s.studentcode, v.PREFIXNAME, s.studentname, s.studentsurname, 
         v.FACULTYNAME, v.departmentname, s.studentyear, v.SEX, 
         b.bytedes, s.studentid, s.studentstatus, e.grade
order by s.studentcode;
```
