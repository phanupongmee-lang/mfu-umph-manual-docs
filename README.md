# 📖 คู่มือการทำงานของอั้ม (Aum's Handover Manual)
**ระบบลงทะเบียนเรียนและถอนรายวิชา มหาวิทยาลัยแม่ฟ้าหลวง (MFU Registration & Withdraw API)**

สวัสดีครับ! โปรเจกต์นี้คือแหล่งรวบรวม **Core Business Logic** (PL/SQL) ที่อั้มดูแลอยู่ ซึ่งเป็นหัวใจหลักของระบบทะเบียน ตั้งแต่เด็กเริ่มแจ้งความประสงค์, แย่งที่นั่งลงทะเบียน, ไปจนถึงการกดถอนรายวิชาติด W 

เอกสารและโค้ดในนี้ทำไว้เพื่อส่งต่องานให้ทีมงาน หรือเอาไว้ทบทวนความจำว่าระบบแต่ละก้อนทำงานยังไงครับ

## 📂 โครงสร้างไฟล์ในโปรเจกต์ (What's inside?)

โค้ดทั้งหมดถูกแบ่งออกเป็น 3 หมวดหมู่หลักๆ ตามช่วงเวลาของปฏิทินการศึกษาครับ:

### 📦 1. ระบบแจ้งความประสงค์ (Pre-Registration)
เป็นช่วงที่นักศึกษาจัดตะกร้าก่อนลงทะเบียนจริง
* `MIS_API_REG_ENROLL_PRE_ENROLL` : จัดการหน้าเว็บ (Add/Drop ตะกร้า) เช็คตารางสอบชน
* `MAINREGISTER_PREREG` : ด่านตรวจสอบเงื่อนไขก่อนยืนยันสิทธิ์ (Check & Confirm)

### 📦 2. ระบบลงทะเบียนหลัก & เพิ่ม-ถอน (Enrollment)
**ด่านที่เดือดที่สุด!** ทำงานแบบ Real-time แย่งที่นั่งและคิดเงินทันที
* `MIS_API_REG_ENROLL_PKG` : ดึงข้อมูลแสดงหน้าจอ และเช็คที่นั่งว่าง
* `MAINREGISTER_SUBMIT` : สมองกลตัวหลัก! (ตรวจสอบเงื่อนไข 6 ด่าน: สถานะ > ปฏิทิน > ตะกร้า > เช็คชน > คิดเงิน > บันทึก) **ถ้ามีแก้กฎการลงทะเบียน ต้องมาดูที่ไฟล์นี้เป็นหลัก**

### 📦 3. ระบบถอนรายวิชาออนไลน์ (Withdrawal)
ระบบ Workflow อนุมัติการถอนรายวิชา (W) 
* `Withdraw_Student_pkg` : ฝั่งนักศึกษา (กดขอถอน, เช็คประวัติ)
* `Withdraw_Officer_pkg` : ฝั่งคนอนุมัติ (อาจารย์ที่ปรึกษา, อาจารย์ผู้สอน, คณบดี, เจ้าหน้าที่)
* `Withdraw_Master_pkg` : จัดการข้อมูลพื้นฐาน เช่น เปิด-ปิดปฏิทิน, ตั้งค่าสิทธิ์ User

## 🚨 จุดที่ต้องระวัง (Aum's Warning & Pitfalls)

ถ้ามีเคสแปลกๆ ทะลุเข้ามา หรือโปรแกรมเมอร์คนต่อไปต้องเข้ามาแก้โค้ด อั้มขอฝากจุดที่ต้องเช็คไว้ตามนี้ครับ:

1. **ระบบ Bypass (เส้นทางลัด):** ถ้าสงสัยว่าทำไมเด็กบางคนทะลุกฎมาได้ (เช่น สอบชนแต่ลงได้, หน่วยกิตเกินแต่ลงได้) ให้ไปเช็คตาราง `AVSREG.ERROR_ABORT_CONFIG` ก่อนเลย เพราะเจ้าหน้าที่อาจจะปลดล็อคให้เด็กเป็นรายบุคคล
2. **การ Hardcode เงื่อนไขพิเศษ:**
   * ช่วงวิกฤต Covid (ปี 63-64) มีการ Hardcode ลดค่าปรับล่าช้าไว้ใน `MAINREGISTER_SUBMIT` 
   * มีการ Hardcode เช็คสาขาวิชาภาษาจีน (`VDEPT = 1007`) สำหรับวิชาปรับพื้นฐาน
3. **การคืนเงิน (Refund Rate):**
   เวลาเด็กกดลดรายวิชา ระบบจะเช็คช่วงปฏิทินเพื่อคิดเปอร์เซ็นต์คืนเงิน ถ้าแก้ปฏิทินต้องระวังกระทบการเงินด้วย
